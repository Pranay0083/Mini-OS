Mini OS — Shell Layer

How every command flows through the system, what data structures back them, and where the real limits are.

Source file map

Bootstrap sequence (main.c)

Steps

  1. virtual_ram[65536] — static 64 KiB array is the backing store for the entire heap.
  2. mem_init(virtual_ram, 65536) — creates one huge FREE block spanning the region.
  3. kb_init() — puts terminal into raw mode (no echo, no buffering); registers atexit(kb_restore).
  4. scr_init(80, 24) — screen layer ready.
  5. shell_init() → vfs_init() (root inode 0) + task_init() (clear task table).
  6. shell_run() — enters REPL; does not return until exit.
  7. kb_restore() — restores terminal; process ends.

Initial heap (after step 2)

HDR
FREE — 65 512 B usable

One BlockHeader (24 B) + everything else free. The header stores size, is_free=1, next=NULL.

Limitations

  • Heap size fixed at compile time — cannot grow.
  • No signal handling beyond atexit.
  • All state lost when process exits.

REPL loop (shell_run)

Tokenization (str_split)

Example: write notes hello world

Input buffer (mutable):

write \0 notes \0 hello \0 world \0

Spaces replaced by \0. Tokens point into the same buffer — no heap allocation.

tokens[0] → "write"   tokens[1] → "notes"   tokens[2] → "hello"   tokens[3] → "world"

Limitations

  • 16-word cap → words 17+ silently lost.
  • No quote support → cannot have a single argument with spaces.
  • Tab is not treated as a delimiter (only ASCII space).

FREE — remaining

■ header (24 B each)   ■ used data   ■ free

Algorithm

  1. First-fit: walk linked list; first free block ≥ needed size wins.
  2. Split: if remainder ≥ MIN_BLOCK_SIZE, create new free block after.
  3. Free: mark block free; scan list and merge adjacent free blocks (forward coalescing).

Protections

  • NULL pointer → ignored.
  • Double free → detected (is_free already set).
  • Out-of-bounds pointer → bounds check against heap_base/heap_size.

help — flow

Pure output. No state change. Extra args ignored.

echo — flow

echo hello world →

echohelloworld

Loop tokens[1..n-1] with space between → "hello world\n". Bare echo prints blank line.

clear

scr_clear() — ANSI escape. Display reset only; VFS/heap untouched.

Limitations

  • help: Static text; can drift from actual code if new commands added.
  • echo: No quoting; 16-token and 256-byte line limits apply.
  • clear: Only clears display; terminal scrollback may still show history.

ls

Information flow

Only shows children where parent == current_dir. Empty → (empty directory). Counts items at the end.

Limitations

  • No path argument — always lists CWD.
  • Order is inode table index, not alphabetical.
  • No file type icons, permissions, or timestamps.
  • Extra tokens ignored.

touch mkdir

Information flow

  1. Check count ≥ 2 — else usage error.
  2. vfs_find_by_name(tokens[1], current_dir) — duplicate? → error.
  3. vfs_find_free_inode() — linear scan for first !is_used. Full? → error.
  4. Fill inode: name (copy ≤ 32), is_dir flag, parent = current_dir, data = NULL.
  5. Increment superblock.total_files or total_dirs.

Limitations

  • touch a b → only a created; b ignored.
  • Name not validated for / or .. as literal characters.
  • No data allocated yet — just metadata.

read / cat

Flow

Both read and cat are the same function. If data==NULL or size==0 → (empty file).

Limitations

  • Only first filename; extras ignored.
  • Not found / is-directory → error.
  • No pagination, line numbers, or binary-safe mode.

rm

Flow

  1. Lookup by name in CWD.
  2. If directory: scan all inodes for parent == this. If any → "not empty" error.
  3. mem_free(inode.data) if file had content.
  4. Clear inode fields; decrement file/dir count.

Limitations

  • Cannot delete non-empty directories (no rm -r).
  • No confirmation prompt.
  • Immediate — no undo / recycle bin.

How it works

Walks the intrusive BlockHeader linked list. For each block: prints status, total size, data portion (size − header), and raw address.

Limitations

  • Uses printf (stdio), not scr_print — different output channel.
  • Assumes heap initialized.
  • For humans only — not machine-parseable.

Scheduling (visual)

Not parallel. Not timer-driven. Cooperative — tasks advance only when the user finishes a command.

Limitations

  • Cannot see internal counter value.
  • No "paused" or "sleeping" states — only active/inactive.
  • No CPU time or tick count displayed.

startcounter

Lifecycle

Limitations

  • If mem_alloc fails → OOM error, no task created.
  • If alloc succeeds but task table full → task_add returns -1 but allocated state is never freed → memory leak.
  • Multiple startcounter calls → multiple independent counters.

kill

Flow

  1. str_atoi(tokens[1]) converts text → int.
  2. Validate: 0 ≤ id < 8 and slot active.
  3. mem_free(state) — returns heap block.
  4. Clear slot: active=0, tick=NULL, state=NULL.

⚠ str_atoi edge case (important)

str_atoi("abc")  → 0   (no digits found)
str_atoi("3x")   → 3   (stops at 'x')
str_atoi("-1")   → -1  (out of range → error)
str_atoi("")     → 0   (empty → 0)

Typos like kill abc silently target task 0.

exit

Shutdown sequence

Limitations

  • No VFS flush — all files lost.
  • Task table not explicitly cleaned; OS reclaims.
  • Extra arguments after exit ignored.

Closing — demo script & Q&A prep

Suggested live demo order

  1. help → show command box.
  2. mkdir docs → cd docs → touch readme → write readme Hello Mini OS → cat readme
  3. cd .. → ls — show root with docs/.
  4. startcounter → run a few commands → tasks → kill 0 → tasks
  5. memmap → sysinfo → point out heap block changes.
  6. exit

One-sentence summary

"We built a read–parse–dispatch loop on a custom heap with an inode-table VFS and cooperative tasks — all in C, with no standard library for core logic."

Honest limitations to mention

  • No real kernel mode or hardware interrupts.
  • Parser: space-only, 16 tokens, 256 B line.
  • VFS: RAM-only, no persistence, 64 inodes.
  • kill typo can hit wrong task (naive str_atoi).
